# Phase 2 (v1.5.0) Implementation Plan

> **Goal**: Add `--ide` and `--template` parameters to support multiple IDE configs and project types (cli/lib/web/notebook).
>
> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Architecture**:
- Split `templates/` into `common/` (shared) + `cli/` (default) + `{type}/` (extended). `--template` selects the type-specific directory.
- IDE configs filtered via `--ide`: `all` keeps everything, `none` drops all IDE files, specific IDE names keep only that config.
- 100% backward compatible: default behavior (no `--template`) matches v1.1.7.

**Tech Stack**: Python 3.11+, typer, pytest, pathlib

## Batch 1: Template Directory Refactor

### Task 1.1: Create `common/` and `cli/` directories

**Files:**
- Create dirs: `src/harness_init/templates/common/`, `src/harness_init/templates/cli/`
- Move shared files to `common/`: `.gitignore`, `.harness/`, `.github/`, `.pre-commit-config.yaml`, `Makefile`, `README*`, `configs/`, `docs/`, `scripts/`, `tasks/`

**Steps:**
1. `mkdir -p` all needed subdirectories

3. Verify with `tree src/harness_init/templates/ -L 2`
4. **Run tests immediately**: `pytest tests/ -q`
   - Expect 10-15 failures due to hardcoded template paths in tests
   - Document each failing test and its root cause (path assumption change)
5. Commit: `refactor(templates): split into common/ and cli/ directories`

### Task 1.2: Update core logic for multi-source copying

**Files:**
- Modify: `src/harness_init/core.py`
- Modify: `src/harness_init/_templates.py`

**Key changes:**
1. `_get_templates_dir()` returns `templates/cli/`; add `_get_common_templates_dir()` returning `templates/common/`
2. In `_templates.py`, add `common_dir: Path | None = None` to `copy_templates()`; copy `common_dir` first, then `templates_dir` (overwrites on collision)
3. Extract `_copy_template_source()` helper (<=30 lines) to keep functions small
4. `_copy_templates()` in `core.py` passes both `type_dir` and `common_dir`
5. `_setup_project()`, `_create_directories()`, `_create_source_files()` accept `template: str = "cli"`
6. `_create_directories()`: `notebook` type adds `notebooks/` instead of `src/{pkg}/`
7. `_create_source_files()`: `notebook` type skips `src/{pkg}/__init__.py`
8. `init_project()` signature: add `template: str = "cli"`

**Tests to add/update:**
- `test_init_project_uses_common_and_cli_templates`: verify both dirs are sourced
- **Fix all failing tests from Task 1.1**: update hardcoded paths, add `template_dir` mocks if needed
- After all fixes, run `pytest tests/ -q` — must be green before proceeding to Batch 2

**Commit:** `feat(core): support multi-source template copying`
